Playbook

Extra context around the score.

This is not a new analysis engine. It is a reference layer so the website feels broader and more useful without making the main scanner harder to use.

Score first.

Treat the overall read as a baseline snapshot, not a final verdict. One scan matters most when it can later be compared against another.

Category second.

Harmony, angles, ratios, and dimorphism are more useful when they are read as a pattern rather than isolated numbers.

baselinetierpatterntrajectory

Consistency wins.

If you want to know whether changes helped, use similar lighting, distance, and expression for each scan.

Keep it honest.

Extreme angles, filters, and dramatic edits make the site feel better for a minute but worse for comparison later.

Use the dashboard.

Saved scans are where the score gets context. A single result is useful; a timeline is more useful.

Best scan is not everything.

Peak result, average result, and most recent result all tell a different story. Read all three.
